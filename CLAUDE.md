# CLAUDE.md — 轻衡(qingheng)

个人健身减脂 PWA。纯原生 JS 单页 + Service Worker,数据在 localStorage,Gist 同步。**动手前先读 `ROADMAP.md` 的「开发约定与可复用件」**,那里有工具函数清单和架构规矩,别重复造已有的东西。

## 硬约束

- 无构建工具、无框架、无运行时 npm 依赖。第三方轮子必须自托管进 `vendor/`(npm 拿包 → 单文件 → sw.js ASSETS → README 轮子表登记),**禁止 CDN 引用**。
- 顶层 `$('#xx').addEventListener` 必须判空——SW 升级间隙「旧 HTML + 新 JS」会让整个 IIFE 挂掉全页失效(v29 真实事故)。
- 密钥(dsKey/syncToken/gistId)永不上传、永不进导出文件。
- 部署是本地 `python3 -m http.server 5173`,手机走局域网。**改完不需要 git push**,push 仅作备份。

## 每次改完的固定流程(缺一不可)

1. `node --check app.js`
2. 桌面浏览器开 `http://localhost:5173/test.html`,断言全绿(新功能顺手加断言;纯函数经 `window.__qh_test` 暴露)
3. `./bump.sh`(一键改齐 4 处版本号,禁止手改)
4. 让用户在手机上「强制刷新」验收,告诉他验证步骤

## 用户(Jiin)的偏好,来自实际协作

- **中文交流,UI 文案全中文**;回复简洁直接,少铺垫。
- **手机是主设备**。所有交互按移动端标准打磨:弹层要能下拉关闭、有取消按钮、防误触(表单内容要草稿缓存,别让人重填);视觉上的小毛病(闪烁、粘连、滑不动)他都会发现,交付前自己先过一遍。
- **数字要经得起追问**。AI 估算的假设必须写明(如哑铃按单只、杠铃按总重;力量消耗按做组+组间休息算),他会质疑不合理的数值,答案要有依据。
- **不重复造轮子**:先找现成的好轮子(能自托管的),找到就登记进 README 轮子表;网上拿不到再自己写(自绘风格:线性、24 grid、stroke 2、CSS 变量着色)。
- **克制的美术**:少 emoji,直觉交互优先,不过度设计;拒绝伪科学映射(如营养素上人体热力图——已明确用六系统点亮卡替代)。
- **要真心话**:主动指出可优化的地方和不建议做的事,给出诚实的优先级排序,别只报喜。
- **文档强迫症(好的那种)**:部署方式、轮子来源、开发约定、事故教训都要写进文档,避免未来会话重新踩坑——你现在读的这份就是这个要求的产物。
- 功能扩展前先给方案(/plan 风格)等确认;修 bug 优先于新功能。
- 数据安全三层备份见 README;动同步/合并逻辑前先看 test.html 里的 mergeDb 断言。
